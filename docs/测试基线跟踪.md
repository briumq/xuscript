# 测试基线跟踪（v1.1）

本仓库的“基线”指：统一 runner 能稳定跑通并进行断言/对比的用例集合。

## 已纳入基线（统一 runner）
- tests/specs：语言规范/语义行为（RunOnly，自带断言/期望失败标记）
- tests/integration：golden 输出对比（RunAndCompare）
- tests/integration_en：可选英文集成用例（若目录不存在会跳过）
- tests/edge：AST vs VM 一致性（可用环境变量关闭）
- examples：示例程序（golden 输出对比；会自动跳过 examples/**/modules/）

## 期望失败标记（specs 专用）
- `// expect_error:"..."`：期望解析/编译阶段产生错误，且错误信息包含该子串
- `// expect_panic:"..."`：期望运行期失败，且错误信息包含该子串

## 英文化迁移（说明）
- 要求：后续不支持中文关键字与函数名；现有中文单测需统一替换为英文版本
- 范围：仓库内所有 *.xu 测试/示例/基准中的中文关键字/函数名
- 方法：依据 `crates/xu_lexer/src/keywords.rs` 的英文映射统一替换；函数/方法名统一为英文
- 验证：统一 runner 跑通 specs/edge/integration_en
