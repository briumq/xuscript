# 英文测试迁移清单（初版）

## 目标
- 将仓库中的所有 *.xu 测试/示例/基准由中文关键字与函数名统一迁移为英文版本。

## 关键字映射参考（部分）
- 控制流：如果/否则/当/遍历 → if/else/while/for
- 逻辑：非/且/或/是/不是 → not/and/or/is/isnt
- 定义：定义/结构/枚举/返回/引入/匹配/方法扩展 → func/struct/enum/return/import/match/does
- 可见性：内部 → inner
- 模块：使用/作为 → use/as

## 待迁移目录示例
- tests/benchmarks/xu\n- tests/edge\n- tests/integration（与 integration_en 对齐）\n- tests/specs 既有中文文件\n- examples（若存在中文关键字）\n- stdlib（若存在中文函数名）

## 执行策略
- 逐文件替换为英文关键字与英文函数/方法名；保留语义不变\n- 替换完成后使用统一 runner 验证 specs/edge/integration_en 通过\n- 若涉及 expect_* 用例，统一转为英文后暂不纳入基线（runner 不支持）

## 验证
- 本地运行：`cargo test -p xu_runtime -- tests/specs`（或使用现有 CI）\n- 英文关键字验证：保证不再出现中文关键字/函数名
